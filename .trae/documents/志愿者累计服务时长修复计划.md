# 志愿者累计服务时长修复计划

## 一、问题诊断

### 1.1 核心问题

`totalHours`（累计服务时长）在整个代码库中**从未被实际计算和写入**：

```
创建志愿者 → totalHours = 0.0
订单完成时   → 只更新 totalOrders / level / updateTime，漏掉 totalHours
提交评价时   → 同样漏掉 totalHours
读取展示     → 永远读到 0.0
```

### 1.2 缺失的数据

`Orders` 表有 `delivery_time`（送达时间），但没有 `pickup_time`（取餐时间）。要做时长计算必须先补上这个字段。

| 已有 | 缺失 |
|------|------|
| `orders.delivery_time` ← 送达时写入 | `orders.pickup_time` ← 取餐时未写入 |

### 步骤 5：`refreshVolunteerCompletionStats` 计算并更新 `totalHours`

**文件**：`OrderServiceImpl.java` L922-951

```java

        .id(volunteerStats.getId())

        .id(volunteerStats.getId())

```

新增辅助方法：

```java
private BigDecimal calculateDeliveryHours(LocalDateTime pickupTime, LocalDateTime deliveryTime) {
    if (pickupTime == null || deliveryTime == null) {
        return BigDecimal.ZERO;
    }
    long seconds = Duration.between(pickupTime, deliveryTime).getSeconds();
    if (seconds <= 0) {
        return BigDecimal.ZERO;
    }
    return BigDecimal.valueOf(seconds)
            .divide(BigDecimal.valueOf(3600), 1, RoundingMode.HALF_UP);
}
```

## 三、方法签名变更

`refreshVolunteerCompletionStats` 和 `updateToCompleted` 之间需要传递原始订单对象以获取 `pickupTime`。

| 方法 | 变更 |
|------|------|
| `updateToCompleted(Orders)` | 无需改签名，`ordersDB` 已含 `pickupTime` |
| `refreshVolunteerCompletionStats(Long volunteerUserId)` | **需加参数** `Orders completedOrder` 或 `Long pickupTimeSeconds` |
| `refreshVolunteerRating(Long volunteerUserId)` | 无需改签名（评价不涉及本次配送时长计算） |

推荐：给 `refreshVolunteerCompletionStats` 增加 `Orders completedOrder` 参数。
