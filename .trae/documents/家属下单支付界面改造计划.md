# 家属下单模拟支付界面改造计划

## 一、目标

家属提交订单后，不再自动调用支付接口，改为弹出一个模拟支付确认界面，用户可以：
- **点击支付**：调用支付接口，订单进入待调度状态
- **暂不支付**：返回历史订单页，订单保持待支付状态，15分钟后由定时任务自动取消

这样待支付状态和定时任务才能在实际使用中发挥作用。

## 二、当前流程 vs 目标流程

### 当前流程（需改造）
```
提交订单 → 自动调用 payment → 成功/失败 → 跳转历史订单页
```
问题：payment 自动执行，订单几乎不会停留在"待支付"状态，定时取消任务形同虚设。

### 目标流程
```
提交订单 → 关闭结算弹窗 → 弹出支付确认弹窗
    ├── [确认支付] → 调用 payment → 成功提示 → 跳转历史订单
    └── [暂不支付] → 提示已创建待支付订单 → 跳转历史订单
```

## 三、现状分析

- `submitOrder` 创建订单后状态为 `PENDING_PAYMENT(1)`
- `paymentOrder` 支付后状态变为 `TO_BE_SCHEDULED(2)`
- `OrderTask` 每分钟扫描 15 分钟未支付的订单自动取消
- 前端已有 `paymentOrder` API，历史订单页已有"继续支付"按钮

## 四、执行步骤

### 步骤 1：改造 `submitCheckout()` 方法
**文件**：`sky-admin-vue/src/views/user-order/index.vue`

- 移除 `paymentFailed`、`paymentFailedMessage` 相关变量
- 移除 `await paymentOrder(...)` 的自动调用代码
- 提交成功后：关闭结算弹窗、清空购物车、保存订单信息（id、orderNumber、payAmount），打开支付确认弹窗
- **不再自动跳转**历史订单页

### 步骤 2：添加支付确认弹窗的数据属性
- `paymentDialogVisible: boolean` — 控制弹窗显示
- `pendingOrder: { id: number; orderNumber: string; payAmount: number } | null` — 待支付订单信息
- `paying: boolean` — 支付中状态

### 步骤 3：添加支付确认弹窗模板
- 在结算弹窗 `<el-dialog>` 之后新增支付确认弹窗
- 显示内容：订单号、待支付金额
- 两个按钮：
  - "确认支付" → 调用 `paymentOrder()` + loading 状态
  - "暂不支付" → 关闭弹窗，跳转历史订单
- 样式风格与结算弹窗保持一致的社区助餐系统风格

### 步骤 4：添加支付处理方法
- `handleConfirmPayment()`：调用 `paymentOrder()` 支付订单
  - 成功：提示"支付成功，订单已进入待调度"，跳转历史订单
  - 失败：提示错误信息，保持弹窗不关闭
- `handleSkipPayment()`：关闭弹窗，提示"订单已创建，请在15分钟内完成支付"，跳转历史订单

### 步骤 5：确保支付失败时的体验
- 支付失败时不关闭弹窗，用户可重试支付或选择暂不支付
- 助餐点休息时提示"当前助餐点已休息"

### 步骤 6：编译验证
- 后端编译通过确认无影响

## 五、不影响的内容

- 后端接口零改动
- 历史订单页"继续支付"按钮保持不变
- 定时任务保持不变
- 操作员、管理员端不受影响
