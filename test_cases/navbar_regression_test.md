## 测试用例 1：退出登录按钮可见性测试

### 测试目标
验证退出登录按钮在登录状态下可见且可点击

。

### 前置条件
1. 后端服务已启动（端口 8080）
2. 前端服务已启动（端口 8888）
3. 已成功登录系统

### 测试步骤
1. 打开浏览器访问 http://localhost:8888
2. 使用管理员账号登录（admin/123456）
3. 登录成功后，观察页面右上角用户名按钮
4. 鼠标悬停在用户名按钮上
5. 观察是否出现下拉菜单
6. 检查下拉菜单中是否包含"修改密码"和"退出登录"选项
7. 点击"退出登录"按钮
8. 观察是否跳转到登录页面

### 预期结果
1. 用户名按钮正常显示
2. 鼠标悬停后，下拉菜单正常显示（不被遮挡）
3. 下拉菜单包含"修改密码"和"退出登录"两个选项
4. 点击"退出登录"后，成功跳转到登录页面
5. Cookie 中的 token、user_info、role 被清除

## 测试用例 3：营业状态显示测试（Redis 有数据）

### 测试目标
验证营业状态能够正确显示（Redis 中已设置 SHOP_STATUS）

### 前置条件
1. 后端服务已启动
2. Redis 服务已启动
3. Redis 中已设置 SHOP_STATUS 键（值为 1 或 0）

### 测试步骤
1. 打开浏览器访问 http://localhost:8888
2. 登录系统
3. 观察页面左上角营业状态显示区域
4. 打开浏览器 DevTools，切换到 Console 标签
5. 查看是否有"营业状态接口返回"的日志
6. 打开 Network 标签，查看 `/shop/status` 请求
7. 验证请求返回的数据格式和值

### 预期结果
1. 页面左上角显示营业状态标签
2. 如果 Redis 中 SHOP_STATUS=1，显示红色"营业中"标签
3. 如果 Redis 中 SHOP_STATUS=0，显示灰色"打烊中"标签
4. Console 中有"营业状态接口返回"日志，且 data.code === 1
5. Network 中 `/shop/status` 请求返回 200 状态码
6. 返回数据格式：`{ code: 1, data: 1 或 0, msg: "..." }`
