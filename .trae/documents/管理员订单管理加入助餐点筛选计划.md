# 管理员订单管理加入助餐点筛选计划

## 一、目标

在管理员端（ADMIN）订单管理页面的筛选栏中，新增"按助餐点筛选"的下拉选择框。

## 二、现状分析

### 2.1 后端（无需修改）
- `OrdersPageQueryDTO` 已有 `diningPointId` 字段
- MyBatis XML `OrderMapper.xml` 的 `pageQuery` 已有 `dining_point_id` 的 WHERE 条件
- 后端接口 `GET /admin/order/conditionSearch` 已支持该参数

### 2.2 前端当前缺少
- 前端 API 接口 `AdminOrderQueryParams` 中未定义 `diningPointId` 字段
- 页面筛选栏未添加助餐点下拉组件
- 页面未加载助餐点下拉数据

### 2.3 参考模板
老人档案页面 `sky-admin-vue/src/views/elderly/index.vue` 已有完整的助餐点筛选实现，可直接参照。

## 三、执行步骤

### 步骤 1：前端 API 接口 - 新增 `diningPointId` 参数
**文件**：`sky-admin-vue/src/api/adminOrder.ts`
- 在 `AdminOrderQueryParams` 接口中新增 `diningPointId?: number`

### 步骤 2：前端页面 - 加载助餐点下拉数据
**文件**：`sky-admin-vue/src/views/orderDetails/index.vue`
- 导入 `getDiningPointList` 和 `DiningPointOption` from `@/api/diningPoint`
- 新增 `queryDiningPointOptions: DiningPointOption[]` 数据属性
- 在 `created()` 中调用 `loadQueryDiningPointOptions()` 加载下拉选项

### 步骤 3：前端页面 - 添加助餐点下拉筛选组件
**文件**：`sky-admin-vue/src/views/orderDetails/index.vue`
- 在筛选栏 `.toolbar` 中新增 `<el-select>` 组件，放置在现有筛选条件之后、"查询/重置"按钮之前
- 绑定 `query.diningPointId`，添加 `clearable`、`filterable` 属性
- `@change="handleQuery"` 实现选择即查询

### 步骤 4：前端页面 - 查询方法传递 `diningPointId` 参数
**文件**：`sky-admin-vue/src/views/orderDetails/index.vue`
- 在 `getAdminOrderPage` 调用的参数对象中增加 `diningPointId: this.query.diningPointId || undefined`
- 在 `resetQuery` 方法中重置 `diningPointId`

### 步骤 5：编译验证
- `mvn -q -DskipTests compile` 验证后端编译通过
- 检查前端 TypeScript 类型无误

### 步骤 6（可选）：OPERATOR 角色不展示
- 由于 OPERATOR 订单视图 `OperatorOrderCenter` 已有自己的筛选逻辑，仅在 ADMIN 角色视图模板中添加
- `v-if` 逻辑保持不变

## 四、不影响的内容
- 不修改后端接口和 DTO
- 不修改 MyBatis XML
- 不修改操作员（OPERATOR）视图
- 不影响现有筛选条件（订单号/手机号/日期范围/状态Tab）
